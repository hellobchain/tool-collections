
# 项目需求：智能合同要素提取系统

## 1. 项目背景
当前业务中，合同审核、归档和风险分析需要大量人工阅读和摘录关键信息，效率低且易出错。为提升合同处理效率，需构建一套基于大语言模型（LLM）的**合同要素自动提取服务**，支持用户上传合同文档，系统自动识别并输出指定字段的结构化数据。

---

## 2. 核心目标
- 支持上传 **.doc** 和 **.docx** 格式的合同文档。
- 支持用户**动态配置**需要提取的要素字段（字段名称及描述）。
- 调用 **LLM（大语言模型）** 对文档内容进行语义理解，自动抽取对应字段值。
- 提供**可视化界面**用于上传、配置、启动提取任务及查看结果。
- 提取结果支持**在线校对与修改**，并支持导出为 Excel/CSV。

---

## 3. 功能需求详情

### 3.1 文档上传模块
- 支持单个或批量上传（最多10个文件）。
- 文件格式限制：`.doc`、`.docx`。
- 文件大小限制：单个文件 ≤ 50MB。
- 上传后系统自动解析文档文本内容（保留段落结构、表格、页眉页脚基础文本）。
- 上传进度与状态提示（上传中、解析中、提取中、完成、失败）。

### 3.2 要素字段配置模块
- **预置常用字段模板**（如：合同编号、合同名称、签约双方、总金额、签署日期、有效期、付款方式、违约责任条款等）。
- **自定义字段**：用户可新增/删除/修改字段，每个字段包含：
  - 字段名称（如“甲方公司全称”）
  - 字段描述/提取指引（如“请从合同中提取甲方全称，通常出现在合同首部签约方位置”）
  - 字段数据类型（文本、数字、日期、金额、枚举值等）
  - 是否必填
  - 是否支持多值（如多个收款账户）
- 支持保存常用字段组合为“模板”，供后续任务复用。

### 3.3 LLM 提取引擎
- 对接企业内部或云端 LLM（如 GPT-4、Claude、文心一言、Qwen 等），通过 API 调用。
- **输入构造**：将文档全文 + 用户定义的字段列表 + 字段描述，组装为结构化 Prompt（如 JSON Schema 格式要求输出）。
- **输出解析**：要求 LLM 以结构化格式（如 JSON）返回提取结果，系统解析并校验数据类型。
- **异常处理**：
  - 若 LLM 返回格式错误，自动重试（最多3次）。
  - 若字段缺失或无法识别，标记为“未提取到”，并记录原因（如“未在文中找到相关信息”）。
- 支持字段间依赖逻辑（如“若签约主体为个人，则无需提取法定代表人”）。

### 3.4 结果展示与校验
- 以表格形式展示每个文档的提取结果，行=文档，列=要素字段。
- 支持**在线编辑**单元格，人工修正提取值。
- 对每个字段显示**置信度评分**（由 LLM 返回或系统计算），低置信度项高亮提示。
- 支持**标注反馈**：用户可标注错误类型（如“提取错误”“遗漏信息”），用于后续模型优化。

### 3.5 导出与集成
- 支持将提取结果（含人工修正）导出为 **Excel（.xlsx）** 或 **CSV** 格式。
- 提供 **RESTful API**，供第三方系统调用（上传文档 + 字段配置 → 返回提取结果）。

---

## 4. 非功能性需求

| 类别 | 要求 |
|------|------|
| **性能** | 单文档平均提取时间 ≤ 30秒（文档页数 ≤ 50页）；支持并发处理5个任务 |
| **准确性** | 核心字段（签约方、金额、日期）提取准确率 ≥ 90%（基于测试集） |
| **安全性** | 文档仅临时存储，任务完成后72小时自动删除；支持权限控制（不同用户仅可见自己的任务） |
| **可扩展性** | LLM 服务可热切换（支持配置多套模型，按策略路由） |
| **日志与监控** | 记录每次提取的请求/响应、耗时、Token 消耗；异常告警 |

---

## 5. 用户角色
- **普通用户**：上传文档、配置字段、查看和修正结果、导出。
- **管理员**：管理字段模板、查看系统日志、配置 LLM 参数（如 temperature、max_tokens）、监控配额。

---

## 6. 技术栈建议（仅供参考）
- 后端：Python（FastAPI/Django）或 Java（Spring Boot）
- 文档解析：python-docx、Apache POI、Tika
- LLM SDK：LangChain / LlamaIndex 或直接 HTTP 调用
- 数据库：PostgreSQL（存储任务、结果、模板）
- 缓存：Redis（任务状态）
- 前端：React / Vue3 + Element Plus / Ant Design

---

## 7. 验收标准（简要）
- 可成功上传 doc/docx，解析文本无误。
- 自定义字段可正常保存并用于提取。
- LLM 返回结果能正确解析并展示。
- 导出文件内容与界面展示一致。
- 多任务并发不报错，异常处理符合预期。
