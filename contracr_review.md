# 合同智能审查系统功能需求说明书

## 一、项目背景

随着公司业务规模不断扩大，合同审查工作量持续增长，传统人工审查方式存在效率低、标准不统一、易遗漏风险点等问题。为提升合同审查效率与质量，拟建设合同智能审查系统，借助OCR识别、自然语言处理等技术，实现合同文档的自动化审查与风险预警。

## 二、需求目标

本系统旨在为用户提供一站式的合同智能审查工具，核心功能包括：

1. 支持多格式合同文档的上传与解析
2. 基于可配置的审查规则库，结合合同类型、审查立场、审查标准，自动输出审查意见
3. 提供可视化、可交互的审查报告，辅助审查人员高效完成合同评审工作

**核心价值**：降低人工审查工作量60%以上，提升审查一致性，减少风险漏查。

## 三、功能需求详细说明

### 3.1 文档上传与解析模块

**功能描述**：用户上传待审查的合同文档，系统自动解析并提取结构化文本内容。

| 需求项 | 详细说明 |
|--------|----------|
| 支持格式 | .doc、.docx、.pdf（其中pdf同时支持文字版和扫描版，扫描版需OCR识别） |
| 上传方式 | 支持点击上传、拖拽上传、批量上传（单次最多5份） |
| 文件限制 | 单个文件不超过20MB |
| 解析反馈 | 上传后显示解析进度（读取中、OCR识别中、解析完成/失败） |
| 内容展示 | 解析完成后，页面左侧展示合同原文内容，支持关键词高亮定位 |

**交互要点**：
- 上传区域有明确的状态提示（待上传/上传中/解析完成/解析失败）
- 解析失败时给出具体错误原因提示

### 3.2 审查配置模块

**功能描述**：用户在上传文档后，需配置本次审查的参数，系统据此调用相应的审查规则。

**3.2.1 合同类型选择**

系统内置以下合同分类体系（支持后续扩展），用户单选或可多选：

| 大类 | 子类 |
|------|------|
| 买卖合同 | 设备采购、原材料采购、服务采购、框架协议 |
| 租赁合同 | 房屋租赁、设备租赁、场地租赁 |
| 服务合同 | 技术服务、咨询服务、物业服务、运输服务 |
| 劳动合同 | 劳动合同、劳务派遣、保密协议、竞业限制 |
| 投融资合同 | 股权转让、增资协议、借款合同、担保合同 |
| 工程合同 | 施工总包、分包合同、勘察设计、监理合同 |
| 知识产权合同 | 专利许可、商标转让、版权授权、技术开发 |
| 其他 | 自定义类型（支持用户手动输入） |

**3.2.2 审查立场选择**

用户根据自身角色选择审查立场，系统据此调整审查的侧重点和风险敏感度：

| 立场 | 说明 | 审查侧重点 |
|------|------|------------|
| 甲方立场 | 作为合同甲方（采购方/委托方/出租方等） | 重点审查履约标准、验收条款、付款节奏、违约责任对等性、质保期限 |
| 乙方立场 | 作为合同乙方（供应商/服务方/承租方等） | 重点审查付款节点、验收条件合理性、违约责任上限、知识产权归属 |
| 中立立场 | 作为合同见证/审批方 | 全面审查条款合规性、公平性，关注法律风险 |

> **示例**：同一份采购合同，甲方立场会更关注“验收不通过的后果”和“延期交付的违约金”，乙方立场会更关注“付款条件是否苛刻”和“违约金额度是否过高”。

**3.2.3 审查标准选择**

用户选择本次审查适用的标准，系统据此匹配对应的规则集：

| 标准类型 | 说明 |
|----------|------|
| 内部合规标准 | 公司内部管理制度、审批权限、合同模板规范等（后台可配置） |
| 法律法规标准 | 民法典合同编、相关行业法规、司法解释等 |
| 行业标准 | 根据合同类型自动关联（如建设工程合同关联GB/T 50353等） |
| 自定义标准 | 用户可选择已配置的自定义审查清单（后台预先配置） |
| 综合标准 | 以上三项的综合 |

**3.2.4 配置交互流程**

建议采用向导式（Step by Step）配置界面，分三步引导用户完成配置，减少认知负担：

**步骤一：选择合同类型** → **步骤二：选择审查立场** → **步骤三：选择审查标准**

每步配置完成后，界面实时显示当前配置摘要，并预估审查范围（如“将执行45项审查规则”）。

### 3.3 审查引擎模块

**功能描述**：用户点击“开始审查”后，系统后台根据配置调用审查规则引擎，对合同文本进行自动化审查分析。

**3.3.1 审查规则体系**

审查规则分为以下三个层级，由粗到细逐级展开：

**第一层：规则大类**
- 条款完整性审查 —— 检查必备条款是否缺失
- 合规性审查 —— 检查条款是否违反法律法规或内部制度
- 权利义务对等性审查 —— 检查双方权利义务是否显失公平
- 风险条款审查 —— 识别高风险的条款约定
- 文本规范性审查 —— 检查表述是否清晰、无歧义

**第二层：规则中类**（以“风险条款审查”大类为例）
- 违约责任条款
- 知识产权条款
- 保密条款
- 争议解决条款
- 终止与解除条款

**第三层：原子规则**（以“违约责任条款”中类为例）
- 违约金比例是否超过法定上限
- 是否仅单方约定违约责任
- 违约金计算标准是否明确
- 是否约定了免责条款

**规则示例**：

| 规则ID | 规则名称 | 所属大类 | 触发条件 | 风险等级 | 建议输出 |
|--------|----------|----------|----------|----------|----------|
| R001 | 违约金比例超限检测 | 风险条款审查 | 违约金 > 合同金额30% | 高 | 建议调整至不超过30% |
| R002 | 单方违约责任检测 | 权利义务对等性 | 仅约定一方违约责任 | 中 | 建议增加对等违约责任条款 |
| R003 | 管辖条款缺失检测 | 条款完整性 | 无争议解决/管辖条款 | 高 | 建议补充争议解决条款 |
| R004 | 知识产权归属模糊 | 风险条款审查 | 未明确约定知识产权归属 | 中 | 建议明确归属方及使用权 |
| R005 | 保密期限缺失 | 条款完整性 | 有保密义务但无保密期限 | 低 | 建议约定保密期限 |

**3.3.2 审查流程**

```
用户点击"开始审查"
    ↓
系统读取配置（合同类型 + 立场 + 标准）
    ↓
匹配对应的规则集（后台规则引擎根据配置自动筛选）
    ↓
逐条执行规则审查（NLP解析合同文本 + 规则匹配）
    ↓
汇总审查结果（按风险等级/条款位置分类）
    ↓
生成审查报告
```

**3.3.3 审查进度反馈**

审查过程中需展示实时进度，包含以下信息：
- 总体进度百分比
- 当前正在执行的审查项名称
- 已发现的风险数量（按高/中/低风险分类展示）

### 3.4 审查报告模块

**功能描述**：审查完成后，系统生成结构化的审查报告，支持在线查看、编辑批注和导出。

**3.4.1 报告概览区（页面上方）**

| 信息项 | 说明 |
|--------|------|
| 合同基本信息 | 合同名称、类型、审查时间、审查人 |
| 审查配置摘要 | 所选立场、标准、生效规则数量 |
| 风险统计 | 高风险X项 / 中风险X项 / 低风险X项 / 通过X项 |
| 审查结论 | 综合评级（建议通过/有条件通过/建议不通过）及一句话总结 |

**3.4.2 审查明细列表**

按风险等级（高→中→低）排序展示每条审查意见，每条意见包含：

| 字段 | 说明 |
|------|------|
| 风险等级 | 高/中/低（以色块区分：红/橙/蓝） |
| 所属章节 | 合同中的章节/条款编号 |
| 规则名称 | 触发的审查规则名称 |
| 原文片段 | 命中问题的合同原文（摘录） |
| 问题描述 | 具体问题说明 |
| 修改建议 | 系统给出的参考修改建议或推荐条款表述 |
| 关联法条 | 引用的法律法规依据（如有） |
| 操作按钮 | 标记“已处理”/“忽略”/“添加批注” |

**3.4.3 交互功能**

- **原文定位**：点击任意审查明细中的“原文片段”区域，右侧合同正文区自动滚动定位到对应段落并高亮显示，便于用户对照查看
- **批注与处理**：审查人员可对每条风险标注处理意见（已修改/待确认/忽略/不适用）
- **筛选与搜索**：支持按风险等级、条款章节筛选；支持关键词搜索
- **审查状态同步**：所有批注和处理状态标记实时保存，支持审查人员之间协作留痕

**3.4.4 报告导出**

| 导出格式 | 说明 |
|----------|------|
| Word文档 | 生成带批注意见的审查报告（含封面、概要、明细、附件） |
| PDF | 只读格式，适合存档/盖章/对外发送 |
| Excel | 审查明细清单，适合数据汇总分析 |

导出内容需包含：审查概览 + 完整审查明细 + 统计图表 + 批注记录。

### 3.5 历史记录与追溯

**功能描述**：所有审查记录自动保存，支持查看和追溯。

- 列表展示历史审查记录（合同名称、审查时间、审查人、合同类型、风险总数、综合评级）
- 支持按时间、合同类型、审查人筛选
- 点击可查看历史审查报告（只读模式）
- 支持删除记录（需二次确认）

## 六、验收标准

1. **上传解析**：支持doc、docx、pdf三种格式成功上传并正确解析文本内容，OCR识别准确率≥90%
2. **审查配置**：合同类型、立场、标准选择后，系统正确匹配对应的审查规则集（需提供测试用例验证）
3. **审查执行**：审查完成后正确输出风险列表，风险等级分类准确，无漏查、误查（以人工复核为基准）
4. **报告功能**：审查报告完整展示概览、明细、原文定位，导出文件格式正确、内容完整
5. **性能指标**：单份合同（50页以内）审查完成时间≤3分钟
6. **规则可配置**：后台规则管理页面支持增删改查审查规则，变更后实时生效

|------|----------|
| 前端 | Vue3 + Element Plus / React + Ant Design |

| 缓存 | Redis |

## 九、附录：术语表

| 术语 | 说明 |
|------|------|
| 审查规则 | 系统内置的用于判断合同条款是否合规/存在风险的判断逻辑，由规则条件、匹配逻辑、风险等级、建议输出四部分组成 |
| 审查立场 | 用户基于自身在合同中的角色选择的审查视角，影响同一条款的风险判定倾向 |
| 审查标准 | 审查所依据的规则来源，如内部制度、法律法规、行业规范等 |
| OCR | 光学字符识别，用于将扫描版PDF中的图片文字转换为可编辑文本 |
| 原子规则 | 不可再分的最小审查单元，每条原子规则针对一个具体的审查点 |

## 十. 输出格式

在 API 接口设计中，无论业务处理成功或失败，HTTP 状态码均统一返回 200。响应体结构采用统一格式，具体字段定义如下：

- **code**：业务状态码，类型为整型。当值为 `0` 时表示请求处理成功；其他非零值则表示失败，具体错误类型由不同错误码区分。
- **msg**：结果描述信息，类型为字符串。成功时通常返回 `"success"`，失败时返回对应的错误提示信息。
- **data**：业务数据负载，类型为对象。成功时返回具体的业务数据；失败时通常返回空对象或 `null`。

示例响应结构如下：

```json
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```

这种设计将 HTTP 传输层与业务状态解耦，便于前端统一处理响应，同时通过 `code` 字段精细化区分业务异常。
